# Планирование проекта
## Цель
Спроектировать, разработать и подтвердить качество
платёжного микросервисного решения «OrbitaMarket» для
обработки заказов и списания геокредитов, используя Java,
Spring Boot, Kafka и PostgreSQL.

Основные задачи:
 * Спроектировать архитектуру двух микросервисов (Orders и Payments)
с асинхронной коммуникацией через Apache Kafka;
 * Реализовать конкурентно-безопасное управление балансом с
использованием оптимистической блокировки;
 * Обеспечить effectively exactly-once списание средств по order_id
с применением паттернов Transactional Outbox и Inbox;
 * Разработать API Gateway для единой точки входа и маршрутизации запросов.
 * Покрыть ключевые сценарии автотестами (REST API + Kafka);
 * Провести анализ безопасности (SAST, секреты) и задокументировать
результаты.

## Стейкхолдеры
| Стейкхолдер                         | Роль / Интерес                                                                                                                                                         |
|-------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Пользователи (клиенты)              | Физические и юридические лица, заказывающие продукты и оплачивающие их геокредитами. Ожидают надёжной работы, корректного баланса и прозрачности операций.             |
